2. 管理画面の使い方

ログイン

管理画面（/admin）にアクセスすると、管理キーの入力が求められます。URLにキーをつけて直接アクセスすることもできます：

/admin?key=YOUR_ADMIN_KEY

一度ログインすると、ブラウザに記憶されます。

Dashboard タブ

できること
  • 本日の注文数・売上・累計がひと目で確認できます
  • 営業ON/OFF — OFFにするとお客様は注文できなくなります
  • お知らせ — 「本日売り切れ」などのメッセージをサイトに表示できます
  • 最新の注文 — アクティブな注文が表示されます

注文タブ

注文のフロー
  1. 新規 注文が入る（通知音が鳴ります）
  2. 準備中 「準備開始」ボタンを押す
  3. 出来上がり 「出来上がり」ボタンを押す
  4. お客様に渡したら「受渡済」→ 完了

フィルターで「新規」「準備中」など絞り込めます。

メニュー管理タブ

在庫切れの管理
  • 各メニューのトグルをOFFにすると、お客様のサイトで「売切」と表示されます
  • 仕入れ状況に応じて、営業前に確認してください
  • セットメニュー（makimaki Box / まぐろBox）も個別に管理できます

3. お客様の注文フロー

  1. サイトでメニューを選び、数量を + / - で調整
  2. 「注文する」→ 名前・電話番号・受取時間を入力
  3. カード決済（Stripe Checkout）で支払い
  4. 決済完了 → 店舗案内が表示される
  5. 管理画面に「PAID」として注文が表示される

4. LINE連携

公式LINEのリッチメニューやメッセージに、以下のリンクを貼ることで直接注文ページに誘導できます：

https://makimaki.tokyo/#menu

メニュー・注文セクションに直接ジャンプします。

5. Stripe（カード決済）の設定方法

makimakiのオンライン決済は Stripe を使っています。以下の手順でアカウントを作成し、APIキーを取得してください。

Step 1: Stripeアカウントを作る

  1. Stripe 登録ページ にアクセス
  2. メールアドレス、名前、パスワードを入力して「アカウントを作成」
  3. メール認証を済ませる
  4. ダッシュボードで「ビジネス情報」を入力（事業者名、住所、銀行口座）

個人事業主でもOK。審査は通常1〜2営業日です。

Step 2: APIキーを取得する

  1. Stripeダッシュボード左メニュー →「開発者」→「APIキー」
  2. 「シークレットキー」の「表示」をクリック
  3. sk_live_ で始まるキーをコピー

注意: シークレットキーは絶対に他人に見せないでください。LINEやメールで送らないこと。

Step 3: テストモードで試す（推奨）

  1. ダッシュボード右上の「テストモード」をONにする
  2. テスト用シークレットキー（sk_test_で始まる）を取得
  3. テスト用カード番号: 4242 4242 4242 4242（有効期限・CVC は適当でOK）

テストモードでは実際の決済は発生しません。動作確認が終わったら本番キー（sk_live_）に切り替えてください。

Step 4: キーをサーバーに設定する

設定方法（開発者向け）

ターミナルで以下のコマンドを実行：

fly secrets set STRIPE_SECRET_KEY=sk_live_あなたのキー -a makimaki

設定後、自動的にサーバーが再起動し、カード決済が有効になります。

Stripeダッシュボードでできること

機能場所
売上確認ダッシュボード → 支払い
入金スケジュール残高 → 入金
返金処理支払い → 該当の支払い → 返金
売上レポートレポート → 概要
テスト/本番切替右上トグル

6. レシートプリンター（Star CloudPRNT）の設定

注文が入ると自動でレシートが印刷される仕組みです。Star mC-Print3（WiFi版）を使います。

Step 1: プリンターを購入する

推奨機種

Star mC-Print3（型番: MCP31LB WT JP または MCP31LB BK JP）

  • WiFi対応モデルを選んでください（Bluetooth版はCloudPRNT非対応）
  • 価格: 約4〜5万円
  • 購入先: Amazon / Star公式サイト
  • レシート用紙（感熱紙 80mm幅）も一緒に購入してください

Step 2: WiFiに接続する

  1. プリンターの電源を入れる
  2. 用紙をセットする（フタを開けて感熱紙のロールを入れる）
  3. プリンターの FEED ボタンを長押し（約5秒）→ 設定情報が印刷される
  4. 印刷された紙に IPアドレス が書いてあるので控える（例: 192.168.1.50）
  5. PCまたはスマホのブラウザで http://192.168.1.50（プリンターのIP）にアクセス
  6. 設定画面が表示されたら「Network」→ WiFi設定でお店のWiFiに接続
  7. 「Submit」を押して再起動 → 新しいIPアドレスが割り当てられる
  8. もう一度FEEDボタン長押しで新しいIPを確認

ポイント: プリンターとお店のPCが同じWiFiネットワークにいる必要があります。

Step 3: CloudPRNTを設定する

  1. ブラウザで http://プリンターのIP にアクセス
  2. 左メニューの「CloudPRNT」をクリック
  3. 以下を設定：
項目設定値
CloudPRNT ServiceENABLE
Server URLhttps://makimaki.tokyo/api/cloudprnt
Polling Interval5（秒）
User（空欄でOK）
Password（空欄でOK）
  1. 「Submit」をクリック
  2. プリンターが自動的に再起動

Step 4: makimaki管理画面で有効化する

  1. 管理画面 にログイン
  2. 「プリンター」タブを開く
  3. 「プリンター有効」トグルを ON にする
  4. 「レシートプレビュー」ボタンでテスト

ONにすると、新しい注文が入るたびに自動でレシートが印刷されます。

トラブルシューティング

症状対処
レシートが出ない管理画面の「プリンター有効」がONか確認。プリンターのCloudPRNT設定のURLが正しいか確認
プリンターの設定画面が開けないプリンターとPCが同じWiFiか確認。FEEDボタン長押しでIPを再確認
文字化けするプリンター設定画面の「Character Set」で「Japanese」を選択
二重印刷されるポーリング間隔を10秒に増やす
用紙が出てこない用紙の表裏を確認（ツルツルした面が表）。フタがカチッと閉まっているか確認

ランニングコスト

  • レシート用紙（80mm x 80m）: 約200円/ロール
  • 1ロールで約500枚のレシートが印刷可能
  • 電気代: ほぼゼロ（待機時1W以下）
  • 月額費用: なし（CloudPRNTは無料）

7. 困ったときは

症状対処
管理画面に入れないキーが正しいか確認。ブラウザのキャッシュをクリアして再試行
通知音が鳴らないブラウザの音声自動再生を許可してください（画面を一度タップ）
注文が来ないDashboardで「営業中」がONになっているか確認
カード決済エラーStripeのキーが正しく設定されているか確認
レシートが出ない管理画面「プリンター」タブでON確認。詳細はこちら
