2. ボックスファースト注文システム NEW

お客様もスタッフも、まずボックスサイズを選び、次に中身の巻きを選ぶフローに統一しました。

注文の流れ
  1. ボックスを選ぶ — 2本 / 4本 / 6本 の3サイズ
  2. 中身を選ぶ — メニューからタップして詰める（選んだ順に1, 2, 3...と番号表示）
  3. ボックス完成 — 本数分選ぶと自動で確定
  4. 追加 or 注文 — 「もう1ボックス追加」するか、注文へ進む

料金

項目計算
巻き代各メニューの単価 x 本数の合計
消費税小計の10%
箱代¥200 x ボックス数
合計巻き代 + 税 + 箱代

3. 店舗POS画面（iPad用）NEW

iPadのSafariで makimaki.tokyo/pos?key=管理キー を開き、「ホーム画面に追加」するとフルスクリーンのPOSアプリとして使えます。

iPadへの設置方法
  1. iPadのSafariで https://makimaki.tokyo/pos?key=あなたの管理キー を開く
  2. 共有ボタン（□↑）→「ホーム画面に追加」
  3. 名前を「makimaki POS」にして「追加」
  4. ホーム画面のアイコンからフルスクリーンで起動できます

POS画面の使い方

3つのタブ
タブ機能
注文ボックス選択 → 中身選択 → 現金 or カード決済
注文一覧本日の注文一覧。タップでステータス変更（新規→準備中→完了）
設定管理キー保存、営業ON/OFF、プリンターON/OFF

決済方法

ボタン動作
現金お預かり金額を入力 → おつり自動計算 → 確定で注文登録・レシート印刷
カードSquare端末に金額が自動送信 → お客様がカードをタッチ → 決済完了で自動登録

カード決済にはSquare Terminal（実機）がSquareアカウントに接続されている必要があります。

4. Square決済 NEW

オンライン注文・店舗カード決済ともにSquareを使っています。

Squareでできること
機能場所
売上確認Square Dashboard → トランザクション
入金スケジュールSquare Dashboard → 残高
返金処理トランザクション → 該当の決済 → 返金
端末管理Square Dashboard → デバイス

Square Webhookの設定

オンライン決済の完了通知を受け取るために、Square DeveloperでWebhookを設定してください：

  1. Square Developer にログイン
  2. アプリを選択 → Webhooks
  3. 「Add Endpoint」をクリック
  4. URL: https://makimaki.tokyo/api/square-webhook
  5. Events: payment.completed を選択
  6. 保存

5. セルフオーダー画面（店頭お客様用）NEW

店頭に置いたiPadで、お客様自身がボックスと中身を選べるセルフオーダー画面です。

URL
https://makimaki.tokyo/order

認証不要。お客様がそのまま操作できます。

iPadへの設置

  1. iPadのSafariで https://makimaki.tokyo/order を開く
  2. 共有ボタン（□↑）→「ホーム画面に追加」
  3. 名前を「makimaki」にして追加 → フルスクリーンアプリとして起動
  4. iPadを店頭カウンターに設置（横向き推奨）

お客様の操作フロー

  1. ボックスを選ぶ — 2本 / 4本 / 6本をタップ
  2. 中身を選ぶ — メニューをタップ（選んだ順に番号が表示）
  3. 本数分選ぶと自動でボックス確定 → 追加も可能
  4. 「レジでお会計する」 をタップ
  5. 注文番号が大きく表示 → この番号をレジで伝える

注文は「未払い」としてシステムに登録されます。レジ（POS画面）で決済を行ってください。

POS画面との連動 AUTO

セルフオーダーが入ると、POS画面に自動で通知されます：

  • チャイム音が鳴る
  • 「セルフオーダー #xxxx」のトーストが画面上部に表示
  • 自動で注文一覧タブに切り替わる
  • 未処理件数のバッジが表示される

POS画面は5秒ごとに新規注文をチェックしています。iPadでPOS画面を開きっぱなしにしてください。

推奨iPad配置

iPad画面用途
iPad #1/orderカウンターに置いてお客様が操作
iPad #2/pos?key=xxxレジ横でスタッフが操作（決済・注文管理）

6. 管理画面の使い方

ログイン

makimaki.tokyo/admin にアクセスし管理キーを入力。/admin?key=xxx で直接アクセスも可能です。

Dashboard タブ

  • 本日の注文数・売上・累計がひと目で確認できます
  • 営業ON/OFF — OFFにするとお客様は注文できなくなります
  • お知らせ — 「本日売り切れ」などのメッセージを表示

注文タブ

注文のフロー
  1. 新規 注文が入る（通知音）
  2. 準備中 「準備開始」ボタンを押す
  3. 出来上がり 「出来上がり」ボタンを押す
  4. お客様に渡したら「受渡済」→ 完了

メニュー管理タブ

  • 各メニューのトグルをOFFにすると「売切」表示になります
  • 仕入れ状況に応じて営業前に確認してください

7. お客様のオンライン注文フロー

  1. makimaki.tokyo にアクセス → メニューセクションへ
  2. ボックスサイズを選ぶ（2本 / 4本 / 6本）
  3. 中身の巻きを選ぶ（タップで1本ずつ追加）
  4. 複数ボックスの追加も可能
  5. 「注文する」→ 名前・電話番号・受取時間を入力
  6. Square決済で支払い
  7. 決済完了 → 受取案内が表示
  8. 管理画面・POS画面に注文が表示される

9. レシートプリンター（Star CloudPRNT）

注文が入ると自動でレシートが印刷されます。POS画面からの注文も同様です。

推奨機種

Star mC-Print3（WiFi版: MCP31LB WT JP / BK JP）

  • WiFi対応モデル必須（Bluetooth版はCloudPRNT非対応）
  • 価格: 約4〜5万円
  • レシート用紙: 感熱紙 80mm幅

設定手順

  1. プリンターをお店のWiFiに接続（FEEDボタン長押しでIP確認 → ブラウザでIP入力 → WiFi設定）
  2. ブラウザで http://プリンターのIP → 「CloudPRNT」設定：
項目設定値
CloudPRNT ServiceENABLE
Server URLhttps://makimaki.tokyo/api/cloudprnt
Polling Interval5（秒）
  1. 管理画面 or POS設定タブで「プリンター」を ON にする

レシート内容

レシートには以下が印刷されます：

  • 注文番号、受付/受取時間、お客様名・電話
  • 各メニュー名 x 数量 と金額
  • 合計金額、決済方法（カード済 / 現金 / 未払い）
  • Box情報（Box#1(4本): 赤身鉄火, サーモン... のように表示）

10. 困ったときは

症状対処
管理画面・POS画面に入れないURLに ?key=管理キー が付いているか確認
オンライン決済エラーSquareダッシュボードでアカウント状態を確認
カード端末に金額が出ないSquare TerminalがWiFi接続 & 同じSquareアカウントにログインしているか確認
注文が来ない管理画面/POS設定で「営業中」がONか確認
レシートが出ないPOS設定 or 管理画面で「プリンター」がONか確認。CloudPRNTのURL設定を再確認
通知音が鳴らないブラウザの音声自動再生を許可（画面を一度タップ）
