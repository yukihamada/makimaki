1このシステムでできること

makimaki.tokyo は、細巻き専門店のための Webサイト + オンラインオーダー + 管理システム です。

お客様向け

  • メニュー閲覧 — セット2種 + おこのみ22品、写真付き
  • オンラインオーダー — メニュー選択 → 受取時間指定 → カード決済
  • セット専用予約ページ（/box）— LINEから直接飛ばせる
  • 日英切替 — 外国人のお客様にも対応
  • ミュージックビデオ（/mv.html）— MAKIMAKI DAYS が流れる体験ページ

お店側

  • 注文管理 — 新規 → 準備中 → 出来上がり → 受渡済 のフロー
  • 在庫管理 — メニューごとにON/OFF（売切表示）
  • 営業ON/OFF — ワンタッチで注文受付を停止
  • お知らせ — サイト上部にバナー表示
  • キャンセル＋自動返金 — Stripe経由で即座に返金
  • アクセス解析 — PV、カート追加数、注文数、人気メニュー
  • 売上統計 — 日別注文数・売上、30日間の推移
  • 要望管理 — 改善リクエストの投稿・管理
  • 通知音 — 新しい注文が入ると音が鳴る
  • レシート自動印刷 — Star CloudPRNT対応（プリンター購入後すぐ使える）

3日常の運用（毎日やること）

営業開始時

  1. 管理画面を開く（スマホ/iPad/PCどれでもOK）
  2. 「Dashboard」タブで 営業中をON にする
  3. 「メニュー管理」タブで、今日 在庫がないメニューをOFF にする
  4. 管理画面を開きっぱなしにしておく（新規注文で通知音が鳴る）

営業中

  1. 通知音が鳴ったら「注文」タブを確認
  2. 注文内容を見て 「準備開始」 をタップ
  3. 巻き終わったら 「出来上がり」 をタップ
  4. お客様に渡したら 「受渡済」 をタップ

営業終了時

  1. 「Dashboard」タブで 営業中をOFF にする
  2. 売上を確認（Dashboard or 統計タブ）

4注文の流れ

お客様側

  1. サイト or LINEリンクでメニューを見る
  2. ＋/−ボタンで数量を選ぶ（セットは1個から、単品は2本から）
  3. 「注文する」→ 名前・電話・受取時間を入力
  4. 「カード決済で注文する」→ Stripe決済ページへ
  5. カード情報を入力して支払い
  6. 完了画面（店舗案内 + Google Maps）が表示される

お店側

  1. 管理画面に通知音 🔔
  2. 注文カードが表示される（PAID = カード決済済み）
  3. ステータスを順番に更新して管理
  4. レシートプリンター設置済みなら自動で紙が出る

注文ステータスの流れ

6Stripe（カード決済）の設定

現在、Stripeの本番キーが設定済みで カード決済は動作しています。

やっておくべきこと

  1. Stripe Webhookの登録（重要）
    • Stripeダッシュボード → Webhooks
    • 「エンドポイントを追加」をクリック
    • URL: https://makimaki.tokyo/api/stripe-webhook
    • イベント: checkout.session.completed を選択
    • これにより、お客様がブラウザを閉じても決済が確実に記録されます
  2. Stripeダッシュボードの確認
    • 支払い一覧 — 入金確認
    • 残高 — 銀行口座への入金スケジュール

詳しいStripeの設定方法は スタッフガイド に記載しています。

追加でできること（要設定）

注文が入ったらLINEに通知を送る機能を追加できます。必要なもの：

  1. LINE Developers Console でMessaging APIチャネルを作成
  2. Channel Access Token を取得（Messaging API設定の中にある）
  3. オーナーの User ID を取得（OAにメッセージを送るとWebhookで取得できる）
  4. 以下のコマンドで設定：
fly secrets set LINE_CHANNEL_TOKEN=取得したトークン LINE_OWNER_USER_ID=Uから始まるID -a makimaki

設定すると、注文のたびにオーナーのLINEに「新規注文: #xxxx、〇〇様、¥△△△」のようなメッセージが届きます。

10さらに良くするアイデア

短期（1〜2週間）

  • メニュー写真の追加 — 実物の断面写真をメニューカードに表示。コンバージョンが大きく上がる
  • LINEリッチメニュー — 「メニューを見る」「セット予約」「お店の場所」の3ボタンを設定
  • Instagramストーリー連携 — プロフィールリンクに makimaki.tokyo/box を設定
  • 注文確認メール — Resend等でメール送信。注文番号・受取時間のリマインド

中期（1〜2ヶ月）

  • リピーター機能 — 電話番号で注文履歴を紐づけ。「前回と同じ注文」ボタン
  • 時間帯別在庫管理 — 「14:00のmakimaki Boxは残り5個」のようなリアルタイム在庫表示（バックエンドは実装済み）
  • 予約カレンダー — 翌日以降の予約も受け付け
  • UberEats/出前館連携 — デリバリーサービスとの連携
  • ポイント/スタンプカード — LINE上でデジタルスタンプカード

長期（3ヶ月〜）

  • 多店舗展開 — 店舗ごとの管理画面・在庫・売上の分離
  • サブスク — 月額プランで週2回お届け、など
  • 法人向けケータリング — 大量注文の専用ページ・見積り機能
  • アプリ化 — PWA or ネイティブアプリ。プッシュ通知でリピート促進
